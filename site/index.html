Why it matters

對話越長，不代表 AI 越懂你

真正需要帶走的通常不是全部聊天，而是「做到哪裡、為什麼這樣決定、接下來做什麼」。把這些整理成可讀的進度檔，新視窗就不必重讀整段歷史。

🪟

同模型，換新對話

Claude 換 Claude、Codex 換 Codex。離開過長的 context，保留真正有用的工作狀態。

⇄

跨模型，按長處交班

一邊研究與整理，另一邊實作或審查；也能輪流使用兩邊訂閱額度。哪個模型適合哪一步，由你決定。

No-download setup

三步驟安裝，全程跟 AI 說

你不用到 GitHub 下載 ZIP，也不用解壓或搬檔。只要先開一個安全資料夾，再把固定版本的安裝指令貼給 Claude Code 或 Codex。

⚠️
第一次請用新資料夾測試。不要直接用含病歷、未去識別化個資、密碼、金鑰，或其他敏感內容的資料夾。
1

建立一個空白測試資料夾

不要直接在桌面根目錄或家目錄貼上。照你的電腦做：

Windows
  1. 打開「檔案總管」
  2. 左邊點「文件」
  3. 右鍵新增資料夾
  4. 命名「AI進度接力測試」
macOS
  1. 打開 Finder
  2. 左邊點 Documents
  3. 新增檔案夾
  4. 命名「AI進度接力測試」

接著在 Claude Code 選 Local / Select folder，或在 Codex 新增 local project，選剛才的資料夾。

2

把這一句貼給其中一個 AI

Claude Code 或 Codex 選一邊貼一次即可。AI 會抓固定版本、核對雜湊，再直接安裝；完整提示詞不用貼到第二邊。

請把 https://raw.githubusercontent.com/Nouischen/cross-ai-progress-bridge/v1.0.0/INSTALL.md 下載到系統暫存資料夾，確認 SHA-256 為 62B052F6791BDA02C6749B3FD8FEDF320C3A4EE969EC8D32C432E438032C4FA2，完整讀取後，在目前專案逐條執行。不要只說明步驟，請直接安裝；無法完整讀取或雜湊不符就停止，不要猜測或改用其他來源。

看到權限確認時，請先看清楚它只在目前測試資料夾與系統暫存區操作，再允許。若找不到 Git 或目前位置不適合，安裝器會用白話告訴你下一步，不會硬裝。

3

用兩個全新對話驗證

安裝完成先顯示 NOT READY 是正常的。它不會在同一個安裝視窗假裝驗證成功。

全新 Claude Code 對話，第一句輸入驗證接力
全新 Codex 對話，第一句輸入驗證接力
兩邊都完成，任一邊輸入完成驗證

看到 READY，才代表兩邊真的指向同一個實體專案資料夾。

Compatibility

像插座一樣加上去，但不亂碰原系統

相容不是一句「保證不衝突」。真正安全的做法，是先辨識、取得同意，再沿用原本的路徑與狀態語意。

已經有自己的簡單系統？

安裝器先只看檔名。發現 `CLAUDE.md`、`AGENTS.md`、progress、tasks、notes/current 或類似來源時，會列出準備讀取的檔案並問你是否允許整合。

1️⃣

不先偷讀內容探索階段只看名稱；私密筆記、憑證與病歷不在讀取範圍。

2️⃣

不搬家、不蓋掉沿用既有路徑、格式與封存方式，修改前先備份。

3️⃣

對不起來就停既有 status 無法可靠對應 active／blocked／done 時，交回給你決定，不自行發明。

Local & auditable

資料留在你的資料夾，規則公開可查

這個專案沒有後端、帳號、資料庫或遙測。GitHub 提供的是安裝規格；你的任務內容不會回傳給作者。

你會得到什麼

  • 純文字 Markdown，任何編輯器都能打開
  • 固定版本 URL＋SHA-256 完整性檢查
  • 修改既有檔案前備份
  • 偵測同時變更時保留 conflict，不覆蓋原檔
  • Claude Code 與 Codex 的 fresh-session 證據

不要放什麼

  • 未去識別化病歷或病人資料
  • 身分證號、電話等敏感個資
  • 密碼、API key、憑證與私鑰
  • 你不希望任何 AI 讀取的私密筆記
  • 需要兩個 AI 同時編輯的任務
FAQ

先把幾個容易誤會的地方說清楚

一般 ChatGPT 或 claude.ai 網頁聊天框可以用嗎？

不行。這套系統需要 Claude Code 與 Codex 能讀寫同一個本機專案資料夾。一般網頁聊天沒有共同的本機檔案空間。

為什麼第一次不建議在 Git 專案測？

Claude Code Desktop 對 Git 專案可能建立隔離 worktree；如果 Claude Code 和 Codex 指向不同 checkout，就算看起來是同一個專案，也不會讀到同一份未提交進度。熟悉 Git 的人可以使用，但要自行確認兩邊是同一個實體 checkout。

它會自動判斷哪個模型比較擅長嗎？

不會。它保存可交班的工作狀態，不替你評比模型。你可以依任務、實際表現、額度與偏好決定何時切換，再把同一份進度交給另一邊。

是不是完全不會衝突？

它會用備份、revision、候選檔與 conflicts 目錄降低覆寫風險，但只支援依序交班。兩個 AI 同時修改同一任務仍可能衝突，因此明確禁止這樣使用。

為什麼安裝後還要跑兩次「驗證接力」？

「檔案已建立」不等於「兩個新 session 都真的會載入」。fresh 驗證要求 Claude Code 與 Codex 各自在全新對話的第一句讀取同一個隨機測試任務，兩邊證據一致後才切成 READY。

下次換視窗，不必再從頭解釋。

Compatibility & privacy

Additive by design

If the installer finds an existing progress system, `CLAUDE.md`, or `AGENTS.md`, it inventories names first and asks for consent before reading or integrating. It preserves existing content, stops when status semantics cannot be mapped safely, and never uploads project data.

Local and auditable

  • Plain Markdown files
  • Version-pinned URL and SHA-256
  • No backend, account, database, or telemetry
  • Backups and conflict preservation
  • Open installation specification

Important limits

  • Use one AI at a time
  • Both tools must open the same physical folder
  • Web-only chats are unsupported
  • Do not store secrets or unredacted personal data
  • Git worktrees require advanced manual checking
